  • user_data (object or None) â Data passed when the compare func is assigned e.g. by Gtk.TreeSortable.set_sort_func()
Returns:

a negative integer, zero or a positive integer depending on whether a sorts before, with or after b
Return type:

int

A Gtk.TreeIterCompareFunc should return a negative integer, zero, or a positive integer if a sorts before b, a sorts with b, or a sorts after b respectively.

If two iters compare as equal, their order in the sorted model is undefined. In order to ensure that the Gtk.TreeSortable behaves as expected, the Gtk.TreeIterCompareFunc must define a partial order on the model, i.e. it must be reflexive, antisymmetric and transitive.

For example, if model is a product catalogue, then a compare function for the âpriceâ column could be one which returns price_of(@a) - price_of(@b).
